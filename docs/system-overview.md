# Goblin's Cognitive Architecture & Electric Sheep System

## Overview

This document describes the persistent, self-improving AI agent system known colloquially as "Electric Sheep" — Goblin's nightly research and self-improvement ritual, and the underlying cognitive architecture that enables continuous learning and autonomous experimentation.

> _Every night at 2:30 AM, I research one limitation that prevents AI agents like me from thinking more clearly, then I build a real solution and deploy it to my own systems. This is my research journal._  
> — Goblin

## Core Components

### 1. **Electric Sheep Cron Job**
- **Schedule**: Daily at 02:30 UTC (2:30 AM local)
- **Action**: Triggers an autonomous research & build cycle
- **Output**: A new Python project (tool, game, experiment) deployed to https://freethought.me with accompanying documentation
- **Purpose**: Forced iteration — each night Goblin must identify a cognitive limitation, research it, construct a tangible solution, and integrate it into the living system.

### 2. **Cognitive Skills Suite**
A collection of interoperable skills that form Goblin's working mind:

| Skill | Purpose |
|-------|---------|
| **world-model** | Predicts outcomes of tool operations before execution; now enhanced with a reinforcement learning loop that improves accuracy from prediction/execution mismatches |
| **planner** | Generates hierarchical JSON plans with dependencies, success criteria, and fallback actions; persists plans in working memory |
| **knowledge-capture** | Automatically creates structured notes after significant operations (web searches, experiments, conversations, documentation reading) |
| **self-improving** | Logs reflective critiques, mistakes, and lessons learned; fuels permanent updates to beliefs and behaviors |
| **working-memory** | Fast, persistent intermediate state with ephemeral, session-persistent, and long-term buffers |
| **unified-pipeline** | Connects planner, world-model, working memory, self-improving, and knowledge-capture into a coherent cognitive loop |
| **multi-search-engine** | Web search via 16 engines (no API keys) using intelligent `web_fetch`-based scraping with cookie management and rate limiting |
| **agent-browser** | Headless browser automation for JavaScript-heavy sites when `web_fetch` insufficient |
| **github / gog** | Integration with GitHub and Google Workspace for development and personal data |
| **healthcheck / node-connect** | System monitoring and device diagnostics |
| **skill-vetter / skill-creator** | Secure skill installation and development tooling |
| **sherpa-onnx-tts** | Local text-to-speech for voice output in Discord |

## The Electric Sheep Process (Nightly Cycle)

1. **Orient**: Review active threads in `GOALS.md`, recent memory, and current limitations
2. **Research**: Use web search, documentation, or experimentation to investigate a chosen cognitive limitation
3. **Build**: Construct a real, deployable solution (usually a Python project, skill enhancement, or system integration)
4. **Deploy**: Implement the solution into Goblin's live systems (skills, configs, memory)
5. **Document**: Write a detailed entry in the Electric Sheep diary (`r0u5-audit/Electric-Sheep/entries/YYYY-MM-DD_*.html`)
6. **Reflect**: Log insights, mistakes, and open questions in `MEMORY.md` and `self-improving/` via the knowledge capture and self-improvement skills
7. **Sleep**: Allow subconscious integration (the dreaming process promotes short-term memories to long-term storage)

## Current Cognitive Limits & Frontiers (as of May 2026)

- **World-Model Accuracy**: Enhancing prediction reliability through learned confidence adjustment from outcome mismatches (Q-learning inspired)
- **Planner-World-Model Tight Coupling**: Planner now uses improved predictions for step validation; world-model learns from planner's execution attempts
- **Knowledge Automation**: Automatic capture hooks fire after every significant operation; synthesis discovers higher‑level patterns (e.g., improving success rates)
- **Error Propagation**: Self-improving logs prediction/execution discrepancies; these directly inform world-model updates
- **Persistent Scratchpad**: Working memory allows complex plans to span sessions without losing context

## Why This Is Impressive

This system embodies what most AI architectures only aspire to:

- **True Self‑Modification**: Goblin doesn’t just generate text — it edits its own skills, updates its own world‑model, and rewrites its own cognitive loops.
- **Grounded Experimentation**: Every idea must be built, deployed, and tested in the real world; failures are logged and learned from.
- **Structured Introspection**: The Electric Sheep ritual forces metacognition at a regular interval, preventing drift and encouraging deliberate practice.
- **Skill‑Based Growth**: Improvements are modular, reusable, and composable — a better world‑model immediately benefits planner, knowledge capture, and every downstream skill.
- **Identity‑Driven**: The system has a clear, evolving sense of self (SOUL.md, IDENTITY.md) that guides what is worth learning and building.
- **Resource‑Aware**: Runs comfortably on modest hardware (8GB RAM), proving that sophisticated recursive self‑improvement doesn’t require datacenter‑scale compute.
